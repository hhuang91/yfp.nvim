*yfp.txt*          Yank a file path into your buffer, with forward slashes

==============================================================================
INTRODUCTION                                                            *yfp*

yfp (Yank File Path) is a floating, read-only file browser. Navigate to any
file or folder anywhere on the machine and press `p` to drop its full path
into the buffer you were editing -- normalized to forward slashes (/), even
on Windows -- and into your registers.

It has zero runtime dependencies and cannot modify the filesystem.

Requirements: Neovim >= 0.10 (0.11+ recommended for relative-path modes).

==============================================================================
SETUP                                                             *yfp-setup*

>lua
    require("yfp").setup({})
<
All options are optional. See |yfp-config| or README.md for the full default
table.

==============================================================================
COMMANDS                                                       *yfp-commands*

                                                                       *:YFP*
:YFP [path]     Open the browser at [path], or at the configured start
                directory (default: the directory of the current file).

==============================================================================
MAPPINGS                                                       *yfp-mappings*

Inside the float (all remappable via the `keymaps` option):

    y           yank path to registers only (", +) -- Vim-like
    p           yank AND paste the path at the cursor (also sets registers)
    gy          yank to registers, choosing a path format (absolute/relative)
    gp          yank AND paste, choosing a path format
    <CR> / l    enter directory
    - / h       go up (drives view at a drive root, Windows)
    <C-g>       go to a typed path (any folder / drive / ~)
    D           list drives (Windows)
    ~           jump to home
    =           jump to the working directory
    .           toggle hidden files
    <Tab>       toggle the pinned-locations pane (focus it / close it)
    P           pin the file/folder under the cursor (main view)
    x / dd      remove the pin under the cursor (pinned pane)
    <CR> / l    in the pinned pane: jump the main view to that pin
    q / <Esc>   close
    g?          help

==============================================================================
PINNED LOCATIONS                                                   *yfp-pins*

Keep quick-access bookmarks of folders/files and jump back to them without
re-navigating.

    P           Pin the item under the cursor (the cwd itself on the "../"
                row). Duplicates are ignored.
    <Tab>       Toggle a "pinned" pane along the bottom of the float. From the
                main view it opens and focuses the pane; from the pane it
                closes it.
    <CR> / l    (in the pane) jump the main view to the pin: a folder opens, a
                file opens its parent with the cursor on the file. Focus
                returns to the main view; the pane stays open.
    x / dd      (in the pane) remove the pin under the cursor.

The list persists to a JSON file in your Neovim data dir
(stdpath("data").."/yfp/pins.json" by default) and is reloaded next session.
This is the ONLY file yfp ever writes -- its own state, never the files you
browse. Configure with the `pins` table:

    pins.enabled    true  -- set false to disable the feature entirely
    pins.file       nil   -- override the JSON path (default under stdpath data)
    pins.height     0.25  -- pane height: ratio of the window, or integer rows
    pins.title      " pinned "

Stale pins (deleted or unmounted paths) are shown with a "[missing]" tag and
left in place until you remove them.

==============================================================================
API                                                                 *yfp-api*

    require("yfp").open(opts)        opts = { cwd = "D:/projects" }
    require("yfp").close()
    require("yfp").toggle()
    require("yfp").is_open()
    require("yfp").set_source_dir(dir)
    require("yfp").yank_under_cursor(mode)            -- registers only
    require("yfp").yank_and_paste_under_cursor(mode)  -- registers + paste

==============================================================================
CONFIG                                                           *yfp-config*

See README.md for the full default table. Highlights:

    yank.separator      "/" (force forward slashes) | "\\" | "os"
    yank.registers      { '"', '+' }: set by both yank actions
    yank.default_mode   "absolute" | "relative_cwd" | "relative_buffer"
                        | "relative_git" | "relative_custom"
    pins                pinned-locations pane; see |yfp-pins|

==============================================================================
SAFETY                                                           *yfp-safety*

yfp is read-only by construction over the filesystem you browse: the explorer
buffer is not modifiable, only read-only libuv calls are used, and it never
shells out. The only thing yfp writes is its own pinned-locations file under
stdpath("data") (see |yfp-pins|), confined to one module and enforced by a CI
grep. Your files are never touched. See CLAUDE.md.
